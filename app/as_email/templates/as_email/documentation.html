Your Email Accounts

You may have one or more email accounts on this server, each with its own email address. All of your email accounts are listed on the main page after you log in.

Each email account has its own password used to authenticate for both sending (SMTP) and receiving (IMAP) on this server. This password is separate from your login password for this web interface. Set or change an email account's password from its card on the main page.

Sending Email

Configure your email client with one set of SMTP credentials per email account. Use the email address and its account password (not your web login password).

Local Delivery

Local delivery stores incoming mail in directories on this server. This is the simplest delivery method and requires no external accounts or services.

This server also runs an IMAP service that provides access to those local mail directories. To read your email directly from this service, configure your email client to connect via IMAP using your email account's address and password:

IMAP Delivery

IMAP delivery forwards incoming mail received by this service to a remote IMAP server of your choice — Gmail, Fastmail, or any hosted or self-hosted server. Each message is deposited via the IMAP protocol directly into that remote account. Configure IMAP delivery per email account on the main page.

  • Set the remote IMAP server hostname, port, username, and password.
  • Normal mail goes to the inbox.
  • When spam auto-filing is enabled, spam goes to the remote junk folder (the server's designated junk folder is used automatically; if none exists, a folder named Junk is used; otherwise mail falls back to the inbox).

Credentials and re-enabling

When you save an IMAP delivery method with a password, the service immediately tests the connection. The save is rejected if the credentials are invalid, so you always know your settings are correct before they take effect.

The same test runs when you re-enable a previously-disabled IMAP delivery method. If your credentials have changed since the method was disabled, update the password at the same time as you re-enable it.

Automatic retries and failure handling

If delivery to your remote server fails (e.g. the server is temporarily unreachable), the service retries automatically. Retries are spaced out with an exponential backoff — starting at 10 minutes, doubling each time, up to a maximum of 4 hours between attempts.

If delivery cannot be restored within {{ settings.DELIVERY_RETRY_DAYS }} days, the delivery method is automatically disabled and you are notified by email at {{ request.user.email }}. You can re-enable it from the main page once the problem is fixed.

If delivery fails because of an authentication error (wrong password or username), the method is disabled immediately rather than retrying, since retrying will not fix a credential problem. You will receive an email notification explaining which method was disabled and why.

Alias Delivery

Alias delivery lets you receive email sent to multiple addresses and have it all delivered to a single email account. For example, if you have a primary account {% if email_accounts %}{{ email_accounts.0 }}{% else %}alice@example.net{% endif %} and a second account {% if email_accounts|length > 1 %}{{ email_accounts.1 }}{% else %}info@example.net{% endif %}, you can set the second account to alias to the first — so mail sent to either address arrives in the primary inbox.

This is the recommended way to consolidate several email addresses without managing separate inboxes for each one.

Spam Filtering

Each email account has a Scan incoming email for spam toggle (enabled by default). When on, incoming messages are run through SpamAssassin before delivery and X-Spam-* headers are added, including a score in X-Spam-Status. When off, any spam headers injected by the upstream provider are left untouched and no local scan is run.

You can configure what happens to messages identified as spam per email account:

  • Do nothing — deliver as normal regardless of spam score
  • Auto-file — move spam to a designated spam folder

SpamAssassin uses a Bayesian filter that can be trained over time to improve accuracy for your specific mail patterns.
